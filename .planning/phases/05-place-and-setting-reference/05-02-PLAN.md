<interfaces>
**Beat candidate rule (lock for implementation):** Walk the full `OutlineIndex` tree recursively (include `active: false` nodes). For each `OutlineNode`, compute `searchableText` by joining `title`, `description`, and all `customFields` values with newline characters. Let `fn = node.fileName.trim().replace(/\\/g, "/")`. The node is a **chapter-manuscript node** if `fn !== ""` and `/^manuscript\/.+\.md$/i.test(fn)`. **Beat candidates** are nodes where this predicate is **false**. Run word-boundary regex match (same as characters: `\b` + escaped term + `\b`, case-insensitive) for each place `name` and `aliases[]` against `searchableText`. Maintain `referencedInBeats` as a **sorted unique** list of matching **node.id** strings; remove IDs that no longer match on subsequent scans.

**Manuscript scan:** Copy `scanManuscriptFileForCharacters` logic from `src/characterStore.ts` into `scanManuscriptFileForPlaces` using `listPlaces`, `savePlace`, and `referencedByNameIn` only (do not mutate `referencedInBeats` in this function).
</interfaces>
