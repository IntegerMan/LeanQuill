  <name>Task 1: Create beat editor with save-back sync</name>
  <files>src/beatEditor.ts</files>
  <action>
    Create `src/beatEditor.ts` — manages beat markdown files in `.leanquill/beats/` per D-17, D-18, D-20.

    1. `openBeatInEditor(rootPath: string, beatId: string, safeFs: SafeFileSystem): Promise<void>`:
       - Read OutlineIndex, find beat by ID
       - Generate beat markdown content per D-18:
         ```
         ---
         beatId: "{beatId}"
         title: "{beat.title}"
         ---

         {beat.description}
         ```
       - Write to `.leanquill/beats/{beatId}.md` via SafeFileSystem (mkdir beats/ if needed)
       - Open the file with `vscode.window.showTextDocument(vscode.Uri.file(filePath))` in a side column
       - The file is a real file within SafeFileSystem boundary — standard VS Code save works

    2. `syncBeatFromFile(rootPath: string, filePath: string, safeFs: SafeFileSystem): Promise<boolean>`:
       - Called when a file in `.leanquill/beats/` is saved
       - Read file content, parse YAML frontmatter to extract `beatId`
       - Extract body (everything after the second `---` line)
       - Read OutlineIndex, find beat by ID, update `description` field with body content
       - Write updated OutlineIndex
       - Return true if sync succeeded, false if beat not found

    3. YAML frontmatter parsing — simple: split by `---`, extract key-value pairs from first block. Do NOT use a YAML library — the frontmatter is trivial (2 fields, no nesting). Parse beatId and title with regex.

    4. Per D-19: custom key-value fields are NOT in the markdown document. Only `description` body syncs between the file and JSON. All structured metadata lives exclusively in the webview.

    5. Per D-20: JSON (outline-index.json) is the single source of truth. The beat markdown file is a view that syncs its body back to JSON on save.

    Export `openBeatInEditor` and `syncBeatFromFile`.
  </action>
