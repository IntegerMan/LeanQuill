  <name>Task 1: Create OutlineTreeProvider with hierarchical rendering</name>
  <files>src/outlineTree.ts</files>
  <action>
    Create `src/outlineTree.ts` following the `chapterTree.ts` pattern (EventEmitter, refresh, getTreeItem, getChildren).

    1. Define `OutlineTreeNode` union type:
       - `{ kind: 'part'; data: OutlinePart }` — collapsible, icon: `$(symbol-namespace)`
       - `{ kind: 'chapter'; data: OutlineChapter; parentPartId: string }` — collapsible, icon: `$(file-text)`
       - `{ kind: 'beat'; data: OutlineBeat; parentChapterId: string; parentPartId: string }` — leaf, icon: `$(note)`
       Each variant needs a unique `id` for tree stability.

    2. Create `buildOutlineTree(index: OutlineIndex): OutlineTreeNode[]` pure function:
       - Returns top-level Part nodes. Each Part's children are Chapter nodes. Each Chapter's children are Beat nodes.
       - Deactivated nodes (active === false) get `contextValue` suffix `:inactive` for conditional menu targeting.

    3. Create `OutlineTreeProvider` class implementing `TreeDataProvider<OutlineTreeNode>`:
       - Constructor takes `rootPath: string` and `safeFs: SafeFileSystem`
       - `refresh()` method fires `onDidChangeTreeData`
       - `getTreeItem(element)`: Return TreeItem with:
         - Label: node name/title
         - Description: "(inactive)" suffix for deactivated nodes
         - Icon: per node kind, greyed via ThemeColor for inactive
         - CollapsibleState: Expanded for parts, Collapsed for chapters, None for beats
         - contextValue: `outlinePart`, `outlineChapter`, `outlineBeat` (+ `:inactive` suffix per D-23)
       - `getChildren(element?)`: Root returns parts from outline index. Part returns chapters. Chapter returns beats.
       - Internal `_index: OutlineIndex | null` field, lazily loaded on first getChildren call via readOutlineIndex.
       - `reloadIndex()` method: re-reads outline-index.json and fires refresh.

    4. Export `OutlineTreeNode`, `OutlineTreeProvider`, and `buildOutlineTree`.
  </action>

<task type="auto">
  <name>Task 2: Add TreeDragAndDropController with reorder and promote/demote</name>
  <files>src/outlineTree.ts</files>
  <action>
    Add drag-and-drop support to `src/outlineTree.ts`:

    1. Create `OutlineDragDropController` implementing `TreeDragAndDropController<OutlineTreeNode>`:
       - `dragMimeTypes`: `['application/vnd.code.tree.leanquill.outlineTree']`
       - `dropMimeTypes`: `['application/vnd.code.tree.leanquill.outlineTree']`
       - Constructor takes `rootPath: string`, `safeFs: SafeFileSystem`, `onReorder: () => void` callback (fires tree refresh + outline sync after drop)

    2. `handleDrag(source, dataTransfer)`:
       - Serialize dragged node IDs into the DataTransfer with the custom MIME type
       - Format: JSON array of `{ id: string, kind: string }` objects

    3. `handleDrop(target, dataTransfer)`:
       - Deserialize dragged nodes from DataTransfer
       - Read current OutlineIndex
       - Determine operation based on source kind vs target depth per D-08:
         - Same-level reorder: Move node within its parent's children array to target position
         - Promote: Beat dropped at chapter level → convert to chapter (keep title as name, clear beat-specific fields)
         - Demote: Chapter dropped at beat level → convert to beat (keep name as title)
         - Part↔Chapter: Similar promote/demote with appropriate field mapping
       - Write updated OutlineIndex via writeOutlineIndex
       - Call `onReorder()` callback to trigger tree refresh

    4. Create pure helper `reorderOutline(index: OutlineIndex, sourceIds: {id: string, kind: string}[], targetId: string | undefined, targetKind: string | undefined): OutlineIndex`:
       - Extracts reorder/promote/demote logic into a testable pure function
       - Returns new OutlineIndex with nodes moved
       - Handles edge cases: drop on root (move to top-level part), drop on self (no-op)

    5. Update `OutlineTreeProvider` to store a reference to the controller so it can be passed to `createTreeView()` options in Plan 04.
       - Add `getDragDropController(): OutlineDragDropController` method.

    Note: `createTreeView()` registration happens in Plan 04 (extension.ts wiring). This plan creates the provider and controller classes only.
