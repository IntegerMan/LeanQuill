<task type="auto">
  <name>Task 2: Create PlanningPanelProvider with postMessage message handler</name>
  <files>src/planningPanel.ts</files>
  <action>
    Create `src/planningPanel.ts` — the extension-side panel manager and message handler.

    1. Create `PlanningPanelProvider` class:
       - Constructor: `(extensionUri: vscode.Uri, rootPath: string, safeFs: SafeFileSystem)`
       - `private _panel: vscode.WebviewPanel | undefined`
       - `private _activeTab: string = 'outline'`
       - `private _debounceTimer: NodeJS.Timeout | undefined`

    2. `show()` method:
       - If panel exists and is visible, `panel.reveal()` and return
       - Otherwise create via `vscode.window.createWebviewPanel('leanquill.planningWorkspace', 'Planning Workspace', vscode.ViewColumn.One, { enableScripts: true, retainContextWhenHidden: true, localResourceRoots: [this._extensionUri] })`
       - Generate nonce (16 random hex bytes)
       - Read OutlineIndex via readOutlineIndex
       - Set `panel.webview.html` = `renderPlanningHtml(index, nonce, panel.webview.cspSource, this._activeTab)`
       - Subscribe to `panel.webview.onDidReceiveMessage` → route to handler
       - Subscribe to `panel.onDidDispose` → clear `_panel` reference
       - Store panel reference

    3. Message handler — `_handleMessage(msg: any)`:
       ```
       switch (msg.type) {
         case 'tab:switch':
           this._activeTab = msg.tabId;
           break;
         case 'beat:updateField':
           await this._updateBeatField(msg.beatId, msg.field, msg.value);
           break;
         case 'beat:openInEditor':
           // Delegate to beat editor (Plan 04 wires this)
           await vscode.commands.executeCommand('leanquill.openBeatInEditor', msg.beatId);
           break;
         case 'beat:toggleActive':
           await this._toggleBeatActive(msg.beatId);
           break;
         case 'beat:addCustomField':
           await this._addCustomField(msg.beatId, msg.fieldName);
           break;
       }
       ```

    4. `_updateBeatField(beatId: string, field: string, value: string)`:
       - Read current OutlineIndex
       - Find beat by ID across all parts/chapters
       - Update the specified field (handle both standard fields and customFields)
       - Debounced write: clear existing timer, set new 300ms timer to writeOutlineIndex (D-28)

    5. `_toggleBeatActive(beatId: string)`:
       - Read index, find beat, flip active boolean, write immediately (no debounce for toggles)

    6. `_addCustomField(beatId: string, fieldName: string)`:
       - Read index, find beat, add `fieldName: ""` to customFields, write + refresh panel HTML

    7. `refresh()` method:
       - Re-read outline index, re-render HTML, set panel.webview.html
       - OR use postMessage to send `{ type: 'outline:update', payload: index }` and let webview re-render client-side
       - Prefer postMessage approach to preserve webview scroll position and expanded card state

    8. `dispose()` method — clean up panel and timers.

    9. Helper: `findBeatInIndex(index: OutlineIndex, beatId: string): { beat: OutlineBeat, partId: string, chapterId: string } | undefined` — traverses the tree to find a beat by ID.

    Export `PlanningPanelProvider`.
