    **1. slugifyCharacterName(name: string): string**
    Lowercase, replace non-alphanumeric with hyphens, collapse multiple hyphens, trim leading/trailing hyphens, append `.md`.
    ```typescript
    export function slugifyCharacterName(name: string): string {
      const slug = name.trim().toLowerCase()
        .replace(/[^a-z0-9]+/g, "-")
        .replace(/^-+|-+$/g, "");
      return `${slug || "character"}.md`;
    }
    ```

    **2. parseCharacterFile(fileName: string, content: string): CharacterProfile**
    Extract frontmatter between `---` delimiters. Normalize CRLF to LF first.
    Parse each line in frontmatter:
    - `name:`, `role:`, `description:` → string values (strip surrounding quotes)
    - `aliases:`, `referencedByNameIn:` → collect indented `  - item` lines into string[]
    - Any other key with a string value → customFields
    Body = content after the closing `---`.
    Return sensible defaults for missing fields (empty strings, empty arrays).

    **3. serializeCharacterFile(profile: CharacterProfile): string**
    Build YAML frontmatter:
    ```
    ---
    name: {profile.name}
    aliases:
      - alias1
      - alias2
    role: {profile.role}
    description: {profile.description}
    referencedByNameIn:
      - path1
      - path2
    {customField}: {value}
    ---
    {profile.body}
    ```
    For empty arrays (aliases, referencedByNameIn), output `key: []\n`. Forward-slashes only in paths.

    **4. listCharacters(rootPath: string, config: ProjectConfig): Promise<CharacterProfile[]>**
    Build chars dir: `path.join(rootPath, ...config.folders.characters.replace(/\/+$/, "").split("/"))`.
    Try `fs.readdir` on chars dir; if dir doesn't exist return `[]`.
    Filter `.md` files only, read each, `parseCharacterFile`, collect. Sort by fileName.

    **5. createCharacter(name: string, rootPath: string, config: ProjectConfig, safeFs: SafeFileSystem): Promise<CharacterProfile>**
    Generate slug via `slugifyCharacterName(name)`. Check if file exists; if yes, try `{slug-base}-2.md`, `-3.md`, etc.
    Build charsDir, ensure it exists via `safeFs.mkdir(charsDir)`.
    Template profile: `{ fileName: slug, name, aliases: [], role: "", description: "", referencedByNameIn: [], customFields: {}, body: "" }`.
    Write via `safeFs.writeFile(path.join(charsDir, slug), serializeCharacterFile(profile))`.
    Return profile.

    **6. saveCharacter(profile: CharacterProfile, rootPath: string, config: ProjectConfig, safeFs: SafeFileSystem): Promise<void>**
    Build charsDir, write `safeFs.writeFile(path.join(charsDir, profile.fileName), serializeCharacterFile(profile))`.

    **7. deleteCharacter(fileName: string, rootPath: string, config: ProjectConfig, safeFs: SafeFileSystem): Promise<void>**
    Build full path. Validate `safeFs.canWrite(filePath, true)` first — throw if blocked.
    Then `fs.unlink(filePath)`. Ignore ENOENT (already deleted).
    Import `fs/promises` directly for the unlink only (SafeFileSystem has no `deleteFile`; unlink after canWrite check is the correct pattern).

    **8. scanManuscriptFileForCharacters(manuscriptFilePath: string, rootPath: string, config: ProjectConfig, safeFs: SafeFileSystem): Promise<void>**
    Read manuscript text via `fs.readFile(manuscriptFilePath, "utf8")`.
    Get `relativeManuscriptPath` = forward-slashes path relative to rootPath (use `path.relative` + replace backslashes).
    Load all character profiles via `listCharacters(rootPath, config)`.
    For each profile:
      - Build regex for name + aliases: each term gets `new RegExp("\\b" + escapeRegex(term) + "\\b", "i")` where `escapeRegex` escapes regex metacharacters
      - Test against manuscript text
      - Compute new `referencedByNameIn`:
        - If match: ensure `relativeManuscriptPath` is in array (add if missing)
        - If no match: filter `relativeManuscriptPath` out of array
      - If array changed, `saveCharacter(updatedProfile, rootPath, config, safeFs)`

    Where `escapeRegex(s) = s.replace(/[.*+?^${}()|[\]\\]/g, "\\$&")`.

    For `test/characterStore.test.ts`:
    - Use same `withTempDir` helper as other tests
    - For I/O tests: create a `SafeFileSystem` with `allowPath(charsDir)` called before tests
    - Mock config: `{ schemaVersion: "1", folders: { research: "research/leanquill/", characters: "notes/characters/" } }`
