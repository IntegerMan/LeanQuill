  <action>
    Create `src/researchTree.ts`:

    1. **`parseFrontmatter(content: string): { name?: string; created?: string }`** (exported for testing):
       - Match `^---\n([\s\S]*?)\n---` at start of content
       - Extract `name` via `/^name:\s*["']?(.+?)["']?\s*$/m`
       - Extract `created` via `/^created:\s*["']?(.+?)["']?\s*$/m`
       - Return object with found values (undefined if not matched)

    2. **`ResearchItem` type** (exported):
       ```typescript
       export interface ResearchItem {
         kind: "research";
         filePath: string;
         name: string;
         created: string;
       }
       ```

    3. **`buildResearchItems(researchDir: string): Promise<ResearchItem[]>`** (exported pure-ish function for testability):
       - Read directory with `fs.readdir`
       - Filter to `.md` files only
       - For each file, read content and call `parseFrontmatter`
       - Use frontmatter `name` or derive from filename (strip `.md`, strip trailing date pattern `\-\d{4}-\d{2}-\d{2}$`, replace hyphens with spaces, title-case)
       - Use frontmatter `created` or file stat `mtime` as ISO string
       - Sort by `created` descending (newest first)
       - Return `ResearchItem[]`

    4. **`ResearchTreeProvider` class** implementing `vscode.TreeDataProvider<ResearchItem>`:
       - Constructor takes `researchDir: string`
       - `_onDidChangeTreeData` event emitter pattern (same as ChapterTreeProvider)
       - `refresh()` method fires change event
       - `getTreeItem(item)`: Returns `vscode.TreeItem` with:
         - `label`: item.name
         - `description`: formatted date (e.g., "Apr 5, 2026")
         - `collapsibleState`: None
         - `command`: `{ command: "vscode.open", title: "Open Research", arguments: [vscode.Uri.file(item.filePath)] }`
         - `iconPath`: `new vscode.ThemeIcon("book")`
         - `contextValue`: "research"
       - `getChildren()`: Calls `buildResearchItems(this.researchDir)` when no parent element, returns `[]` for children of items (flat list)

    - Test `buildResearchItems` with: empty dir, mixed file types, multiple .md files with frontmatter, files without frontmatter (fallback naming), sort order
    - Do NOT import vscode — test only the pure functions (`parseFrontmatter`, `buildResearchItems`)
